Paragraph Design for Web (Cont.)

  • Take a look at the web pages your classmates designed and coded for homework last week. You can browse them at the bottom of this page.
  • Choose ONE of your classmates’ web pages to download. This week, you will edit each web page’s code. Your goal is to edit the design of each web page so that despite their formal differences, their is some kind of formal continuitiy between them. They should feel "of a piece."
  • You will also style the landing page. Design it to serve three goals: 1/ There should be formal continuity between it and the pages it links to. 2/ A casual web browser should quickly get a sense of what this web page is and what it links to. You can add headers, copy, etc. to achieve this goal. 3/ The links should feel worth clicking. A casual web browser should feel compelled to click them.
  • Required: Use google fonts. Make sure your name, the class name, the professor's name, and the date are on the landing page.
  • Edit the code: Step one of editing your classmate’s code will be to “clean it up”. That means indenting and spacing the code so it appears more organized and is easier to comprehend. It also means making sure only the content that should be in the <head></head> or <body></body> sections are in those sections. It also means making sure all “opening tags” like a <div>, <p>, or <h1> are eventually followed by closing tags like a </div>, </p>, or </h1>.
  • For inspiration: Flip through the case studies in Designing the Editorial Experience. See a design you like? Copy it.
  • For coding resources: Browse all of different kinds of styling techniques at W3 Schools. I want you to explore new fonts, line-spacing, top and bottom margins. Email me if you cannot find what you need.
  • Don't forget: Lay out and design the quote for mobile. (I emailed you last week with instructions on how to view your page for mobile on Chrome).
  • Your HTML and CSS web files should be saved in a project folder. Label the project folder as your first name (ex: dylan – one word, all LOWERCASE). Label the HTML files exactly like so: index.html (for the landing page), and index-1.html, index-2.html, index-3.html. Label each one’s corresponding CSS page exactly like so: styles.css for the landing page, and styles-1.css, styles-2.css, styles-3.css (and make sure that in the <head> section of each HTML page, it correctly links to its corresponding CSS page).
  • Upload the folder to our shared google drive folder by 10:00AM next Thursday.
